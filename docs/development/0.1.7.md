## Release Note Draft

### Added

- Prompt comments can now be written as line-start `#` comments. Comment lines
  are classified separately from prompt tags and can be highlighted in Prompt
  Studio.
- Prompt Studio comment display can be configured, including whether comment
  lines are italicized.
- Autocomplete can now be configured to commit with `Enter` and `Tab`, or with
  `Tab` only. `Shift+Enter` remains a normal line break.
- Autocomplete can optionally append a trailing `, ` after a committed
  suggestion.
- Autocomplete adds options for natural-language handling:
  `Do not comma-correct after period` and `Detect natural text by periods`.

### Improved

- Autocomplete now preserves the selected suggestion while the input text and
  caret position have not changed, so arrow-key navigation no longer resets to
  the first suggestion.
- Autocomplete insertion uses editable range replacement so committed
  suggestions can be undone with the text editor's normal undo shortcut.
- Autocomplete preserves natural-language sentence text before a period and
  completes only the active tag fragment when sentence detection is enabled.
- Existing separators are reused and normalized. For example, completing after
  `wisteria,dd` with separator append enabled moves the caret to
  `wisteria, |dd` and normalizes the text to `wisteria, dd`.

### Fixed

- Fixed comment parsing edge cases introduced by slash-style comment handling;
  prompt comments now use the safer line-start `#` convention.
- Fixed autocomplete cache/refresh behavior that could re-render unchanged
  suggestions and reset the active item.
- Fixed separator handling so autocomplete does not duplicate an existing
  comma or overwrite text that is already separated by a comma, newline, or
  natural-language sentence boundary.
