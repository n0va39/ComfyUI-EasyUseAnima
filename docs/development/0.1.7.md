# 0.1.7 Development Plan

This file tracks changes intended for the next package version. It is not a
release announcement.

## Scope

- Keep `pyproject.toml` on the next Registry package version so local work does
  not reuse an already-published package version.
- Do not trigger Comfy Registry publishing, create tags, or publish GitHub
  releases from this plan.

## Planned Changes

- Add LoRA Preset path repair for shared workflow/profile data when local LoRA
  folders differ and a unique filename match exists.
- Add row-level and profile-level `FIX` actions in the LoRA Preset UI.
- Add backend route support for LoRA profile path repair.
- Add regression tests for unique filename repair, ambiguous filename handling,
  unresolved missing paths, and existing-path fast paths.
- Add release-suffixed workflow templates for Registry distribution.
- Document release workflow filename and live-sync rules.
