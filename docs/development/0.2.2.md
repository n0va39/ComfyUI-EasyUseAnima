# 0.2.2 Development Notes

## Scope

0.2.2 prepares the first release-ready draft of the compact Anima AiO generator
flow. The main goals are:

- Keep Prompt Studio responsible for prompt editing and prompt data generation.
- Add `Anima Prompt Corrector Simple` for workflows that only need one multiline
  prompt input and one corrected prompt output.
- Load ANIMA diffusion model, VAE, and CLIP separately through `Easy Use Anima Input`.
- Pass one dedicated `easy use anima input` context socket into `Anima AiO Generator`.
- Store expandable generator settings as keyed versioned JSON, not ordered UI
  widgets that can shift when parameters change.
- Keep the default node compact while moving advanced sampler, model patch,
  Highres, Detailer, preview, and save settings into DOM popups.
- Use release defaults that match the current Anima path: first-pass steps `32`,
  sampler `er_sde`, scheduler `simple`, and AuraFlow shift `3.0` as the
  model-recommended default.

## Runtime Design

The AiO generator supports three first-pass sampler modes:

| Mode | Runtime path |
| --- | --- |
| `comfy_ksampler` | ComfyUI KSampler, with optional Spectrum model patches and corrections applied before sampling |
| `spectrum_mod_guidance_advanced` | Direct `SpectrumKSamplerAdvanced` call |
| `spectrum_spd_speed` | Direct `SpectrumSPDKSampler` call |

Integrated Spectrum sampler modes must not also run the regular KSampler model
patch path for the first pass. If Highres or Detailer is enabled, the generator
may still prepare a standalone Mod Guidance patched model for those later stage
samplers.

`AnimaDAVE` is an optional model patch in `Advanced Options`, not a sampler
backend. When enabled, it is applied before the selected first-pass sampler.

`Model Patch / Optimization` controls belong in `Advanced Options` unless the
control changes the first-pass sampler backend itself. KJNodes FP16
accumulation, SageAttention, Torch Compile, AuraFlow shift, and DAVE follow this
rule. `Sampler Details` should keep `Sampler Backend`, Mod Guidance, Spectrum,
and SPD/SPEED controls.

Highres uses the first-pass CFG, sampler, and scheduler by default. Its release
defaults are `scale_by=1.5` and `denoise=0.25`. Highres and Detailer popups use
a single-column layout so long advanced settings scroll vertically instead of
forming cramped two-column grids.

Detailer targets are stored as keyed target dictionaries plus `detailer.order`.
The UI renders the targets as tabs; tab labels are UI metadata stored in
`detailer.face.label` and `detailer.eye.label`. Runtime dispatch still uses the
stable target keys, not the user-facing tab labels.

## Required And Optional Node Packs

- Required for this package: `ComfyUI-EasyUseAnima`.
- Required by the included AiO sample defaults: `ComfyUI-Spectrum-KSampler`,
  `ComfyUI-Image-Saver`.
- Optional feature dependencies:
  - `ComfyUI-Anima-DAVE`: Anima DAVE model patch in Advanced Options.
  - `ComfyUI-KJNodes`: SageAttention, Torch Compile, FP16 accumulation patches.
  - `ComfyUI-Impact-Pack`: SAM3 detailer path through MaskToSEGS and
    DetailerForEach.

The frontend checks optional node availability through ComfyUI `object_info`.
Missing optional features are locked in the DOM settings UI. Queue
preparation also disables missing optional features before writing the hidden
`generation_settings` JSON into the queued prompt, so saved workflows do not
accidentally call absent optional node packs from the UI.

Direct API execution that bypasses the frontend should still fail with a clear
runtime error naming the missing node pack.

## Image Saver Metadata

The AiO Generator saves the final image size but first-pass sampler settings:

- `Steps`, `CFG`, `Sampler`, `Scheduler`, `Seed`, and `Denoise` use the first
  pass sampler settings.
- `Size` uses the final output image resolution after Highres and Detailer.
- Applied `lora_stack` entries are appended to the Image Saver metadata prompt
  as `<lora:name:weight>` tokens so Image Saver can record Civitai LoRA
  resources and weights. With Image Saver `easy_remix` enabled, those tokens are
  stripped from the visible A1111-style prompt after metadata extraction.
