# 0.2.1 Development Plan

This file tracks planned and already-applied changes for the next package
version. It is not a release announcement.

## Version Boundary

- `v0.2.0` is the release baseline for this plan.
- Changes committed after the `v0.2.0` tag are tracked under `0.2.1`.
- Changes intended for a release before that release is tagged stay in that
  planned version's document.
- Changes made after a release tag move to the next version's document.

## Scope

- Use `0.2.1` as the next package marker after the `v0.2.0` release.
- Build `Anima Prompt Studio Advanced v2` around structured prompt data.
- Make future consumers read prompt data by dict keys instead of positional
  output indexes.
- Add explicit artist-field handling so workflows can either keep the text from
  Prompt Studio's artist-tag input fields in the positive prompt or route it to
  separate artist-conditioning nodes.
- Keep the existing `Anima Prompt Studio Advanced` node compatible while v2 is
  introduced.

## Advanced v2 Data Contract Direction

Prompt Studio Advanced v2 should expose a dedicated context-style socket that
carries a structured prompt data dict. Downstream nodes should read by key and
tolerate unknown future keys.

Candidate type:

```text
EASYUSE_ANIMA_PROMPT_DATA
```

The value should be a Python `dict`, not a JSON string, when passed between
ComfyUI nodes. It may still be serialized as JSON inside saved workflow
metadata or hidden widgets when reproducibility requires it.

Initial candidate keys:

```json
{
  "version": 1,
  "prompt": "",
  "negative_prompt": "",
  "metadata_prompt": "",
  "metadata_negative_prompt": "",
  "width": 1024,
  "height": 1024,
  "fields": [],
  "resolution": {},
  "wildcard": {},
  "naia": {},
  "artist": {
    "handling": "inline",
    "conditioning_mode": "none",
    "include_in_positive": true,
    "text": "",
    "weighted_text": "",
    "tags": []
  },
  "artist_mix": {
    "enabled": false,
    "mode": "prompt",
    "base_source": "positive_without_artist_section",
    "start_percent": 0.5,
    "strength_scale": 1.0,
    "artist_prompt": "",
    "artist_count_hint": 0
  },
  "mod_guidance": {
    "enabled": false,
    "negative_enabled": false,
    "quality_tags": "",
    "negative_prompt": ""
  },
  "flags": {}
}
```

Rules:

- Do not make companion nodes depend on output slot positions when a structured
  prompt data dict is available.
- Keep dict values compact and JSON-serializable unless a later explicitly
  typed runtime value is required.
- Include a `version` key and only change semantics through explicit schema
  versioning.
- Keep old outputs or compatibility adapters until existing workflows have a
  migration path.
- Add tests that prove consumers read by dict key, not by positional tuple
  indexes.
- Treat unknown keys as preserved extension data. Nodes that modify prompt data
  should copy through unknown keys unless the node is explicitly filtering.
- Treat missing keys as defaults. This lets older saved workflows load into
  newer nodes without requiring every future field to exist.

## Context-Style Node Model

The dedicated prompt data socket should work like a context node:

- One node can build the dict.
- Other nodes can pass it through, read selected fields, replace selected
  fields, or merge additional data.
- Adding a new prompt-data field should not require adding a new socket to every
  downstream node.
- Downstream nodes should prefer `prompt_data["key"]` or helper accessors over
  tuple indexes.

## Artist Field Handling Plan

Advanced v2 should treat Prompt Studio artist-tag input fields as first-class
structured data.

Primary reference implementation:

```text
D:\ComfyUI\custom_nodes_workplace\ComfyUI-AnimaPromptEditor
```

Reference files inspected for this plan:

- `README.md`: documents artist mix using editor artist sections and UI modes
  `OFF`, `평균`, and `Exact`.
- `nodes.py`: detects artist sections by `category == "작가"` or section name
  matching `작가|artist`, prepares `artist_mix_prompt`, and builds
  `artist_mix` payload.
- `regional_conditioning.py`: consumes `artist_mix` and implements prompt,
  average, exact, composite exact, late exact, average-late-exact, and
  scheduled-average conditioning.
- `prompt_data.py` and `web/js/anima_xy_utils.js`: define XY axes and mode
  validation for artist mix variants.

Terminology:

- In this plan, "artist tag" means text authored in an artist-tag input field
  or an Advanced field whose role/type is `artist`.
- It does not mean "any prompt token that starts with `@`".
- The `@` prefix is an Anima prompt formatting convention and may appear in the
  artist field text, but it must not be used as the source of truth for
  classification or extraction.
- General prompt fields must not be scanned for `@...` tokens to decide that
  they are artist tags.

Reference compatibility target:

- `ComfyUI-AnimaPromptEditor` separates the base positive prompt from artist
  sections by storing `global_prompt` as `positive_without_artist_section` and
  artist field text under `artist_mix.artist_prompt`.
- `granatta000/anima-artist-mixer` remains an external compatibility target
  because it separates `base_prompt`, `artist_text`, and `blend_mode`, then
  returns a positive `CONDITIONING`.

Prompt Studio v2 should not directly depend on that node. The data contract
should be general enough for that node and similar artist-conditioning nodes.

Basic artist handling combo should follow `ComfyUI-AnimaPromptEditor` first:

| UI mode | Compatibility prompt output | Prompt data behavior |
| --- | --- | --- |
| `OFF` | Preserve current behavior: artist-field text is included in the old `positive_prompt` output. | `artist_mix.enabled: false`, `artist_mix.mode: "prompt"`. Conditioning can append `artist_mix.artist_prompt` to `global_prompt`. |
| `평균` / `average` | Remove artist-field text from prompt outputs intended as base prompt. | `artist_mix.enabled: true`, `artist_mix.mode: "average"`. Encode artist variants and blend them by normalized weights. |
| `Exact` / `exact` | Remove artist-field text from prompt outputs intended as base prompt. | `artist_mix.enabled: true`, `artist_mix.mode: "exact"`. Emit exact artist scheduling metadata for sampler-aware paths. |

Extended modes may be preserved in the dict for compatibility with the
reference implementation and XY jobs, but they do not need to be the first UI
surface:

- `prompt`
- `composite_exact`
- `late_exact`
- `average_late_exact`
- `scheduled_average`

Rules:

- Default mode should be `OFF` to preserve current workflow behavior.
- Artist data should come only from artist-tag input fields or `artist` role
  Advanced fields. Do not infer artists from general fields as a runtime
  fallback.
- The implementation may preserve or format `@artist name` text for Anima
  compatibility, but `@` itself is not the detection rule.
- Preserve explicit artist weights, for example `(@artist_one:1.2)`.
- The prompt-data contract should keep `global_prompt` /
  `positive_without_artist_section` separate from `artist_mix.artist_prompt`,
  matching `ComfyUI-AnimaPromptEditor`.
- The old compatibility `positive_prompt` output may still include artist-field
  text when mode is `OFF`.
- When artist mix is enabled, prompt outputs intended as base prompt should omit
  artist-field text and put it under `artist` / `artist_mix`.
- `metadata_prompt` may keep or omit artist-field text only through an explicit
  policy flag. Do not let metadata behavior drift implicitly from positive prompt
  behavior.
- `artist.text` should be a comma-separated string built from artist fields and
  suitable for nodes that accept `artist_text`.
- `artist.weighted_text` should preserve weighted artist syntax when available.
- `artist.tags` should be structured entries, for example:

```json
[
  {
    "tag": "@artist_one",
    "weight": 1.2,
    "source": "artist_field"
  }
]
```

Companion nodes should be able to use:

- `prompt_data["global_prompt"]` or
  `prompt_data["positive_without_artist_section"]` as `base_prompt`.
- `prompt_data["artist_mix"]["artist_prompt"]`,
  `prompt_data["artist"]["weighted_text"]`, or `artist.text` as `artist_text`.
- `prompt_data["artist_mix"]["mode"]` or
  `prompt_data["artist"]["conditioning_mode"]` as a blend or routing hint.

Unpack nodes should expose at least:

- `artist_handling`
- `artist_conditioning_mode`
- `artist_mix_enabled`
- `artist_mix_mode`
- `artist_mix_start_percent`
- `artist_mix_strength_scale`
- `artist_mix_base_source`
- `artist_text`
- `artist_weighted_text`
- `artist_mix_prompt`
- `positive_prompt_without_artist`

Planned companion nodes:

| Node | Purpose |
| --- | --- |
| `Anima Prompt Data Pack` | Build `EASYUSE_ANIMA_PROMPT_DATA` from existing prompt strings, width/height, Mod Guidance booleans, and optional metadata fields. |
| `EASYUSE_ANIMA_PROMPT_DATA Unpack` | Expose the current compatibility outputs from `EASYUSE_ANIMA_PROMPT_DATA` for old workflow sections. |
| `Anima Prompt Data Set` | Replace selected dict keys, for example `prompt`, `negative_prompt`, width/height, or boolean flags. |
| `Anima Prompt Data Merge` | Merge one prompt data dict with another while preserving unknown keys. |
| `Anima Prompt Data Get` | Read a selected key for lightweight debugging or integration nodes. |

`Anima Prompt Data Pack` is the migration bridge. It should accept existing
values that are currently separate outputs or inputs:

- `positive_prompt`
- `negative_prompt`
- `metadata_prompt`
- `metadata_negative_prompt`
- `width`
- `height`
- `use_anima_mod_guidance`
- `use_negative_anima_mod_guidance`
- `anima_mod_guidance_quality_tags`
- `anima_mod_guidance_negative_prompt`
- `artist_handling`
- `artist_conditioning_mode`
- `artist_mix_enabled`
- `artist_mix_mode`
- `artist_mix_start_percent`
- `artist_mix_strength_scale`
- `artist_text`
- `artist_weighted_text`
- optional existing `prompt_data` to update instead of rebuilding from scratch

## Output Compatibility Rules

Output order must be treated as a compatibility contract.

- Do not reorder or remove the current `Anima Prompt Studio Advanced` outputs:
  `positive_prompt`, `negative_prompt`, `anima_mod_guidance_quality_tags`,
  `anima_mod_guidance_negative_prompt`, `use_anima_mod_guidance`,
  `use_negative_anima_mod_guidance`, `metadata_prompt`,
  `metadata_negative_prompt`, `width`, `height`.
- If the existing node receives a prompt-data output, append it after the
  current outputs. Do not insert it before existing sockets.
- Prefer a separate `Anima Prompt Studio Advanced v2` node if the primary output
  shape needs to be redesigned.
- For v2 nodes, keep the dedicated prompt-data output stable. Future additions
  should happen inside the dict, not by inserting new output sockets.
- If a later release must add a physical output socket, append it at the end or
  create a new node class. Do not insert in the middle.
- Do not rename existing socket names in a way that breaks saved workflow link
  restoration.

## Prompt Data Access Helpers

Implementation should centralize dict access so companion nodes behave the same
way:

- `_normalize_prompt_data(value)` should accept `dict`, JSON string, or `None`
  and return a schema-defaulted dict.
- `_prompt_data_get(data, key, default)` should read nested keys safely.
- `_prompt_data_merge(base, updates)` should deep-copy base data, update known
  keys, and preserve unknown extension keys.
- `_extract_artist_prompt_data(fields, handling_mode)` should return cleaned
  positive prompt parts plus structured artist data from artist-role fields
  only, without falling back from general fields or scanning for `@` prefixes.
- `_build_artist_mix_payload(config, artist_prompts)` should mirror the
  `ComfyUI-AnimaPromptEditor` payload keys: `enabled`, `mode`, `base_source`,
  `start_percent`, `strength_scale`, `artist_prompt`, and
  `artist_count_hint`.
- `_prompt_data_from_advanced_outputs(...)` should build prompt data from the
  existing separate prompt strings, booleans, and resolution values.
- `_advanced_outputs_from_prompt_data(data)` should return compatibility values
  in the old output order.

## Migration Strategy

1. Add the `EASYUSE_ANIMA_PROMPT_DATA` type constant and normalization helpers.
2. Add unit tests for defaulting, unknown-key preservation, JSON/dict input
   handling, boolean fields, and compatibility output ordering.
3. Add `Anima Prompt Data Pack` and `EASYUSE_ANIMA_PROMPT_DATA Unpack`.
4. Add `ComfyUI-AnimaPromptEditor`-compatible artist mix payload construction,
   then artist parsing and prompt separation tests before wiring the UI combo.
5. Add dict modification nodes after the pack/unpack bridge is stable.
6. Add `Anima Prompt Studio Advanced v2` or append a prompt-data output to the
   existing Advanced node, depending on compatibility risk after tests.
7. Update examples only after saved workflow reload and queue validation pass.

## Regression Risks

- Inserting an output before existing Advanced outputs can shift saved links and
  break old workflows.
- Replacing plain outputs too early can break workflows that still connect
  prompt strings and boolean Mod Guidance flags directly.
- Flattening prompt data into a single string removes the reason for the
  dedicated socket and makes future additions brittle.
- Mutation nodes that rebuild dicts from scratch can drop unknown future keys.
- Missing default handling can make old saved workflows fail when new keys are
  added.
- Extracting artist data from general prompt fields or `@` prefixes can hide
  CSV/source data errors and contradict the field-role contract.
- Separate artist mode can accidentally duplicate style influence if
  artist-field text remains in `prompt` and is also sent to an
  artist-conditioning node.
- Exact artist mix metadata can affect sampler compatibility. Preserve a clear
  flag in prompt data so sampler-side code can downgrade or route incompatible
  sampler modes the way `ComfyUI-AnimaPromptEditor` does.
