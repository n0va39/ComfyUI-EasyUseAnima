# 0.2.0 Development Plan

This file tracks planned changes for the next package version. It is not a
release announcement.

## Scope

- Use `0.2.0` as the next package marker after the `v0.1.9` release.
- Add a new regional Prompt Studio node.
- Define regional prompting as mask-scoped prompt application: user-authored
  masks choose where specific prompt fields apply.
- Keep the existing `Anima Prompt Studio Advanced` node compatible.
- Use `D:\ComfyUI\custom_nodes_workplace\ComfyUI-AnimaPromptEditor` as a
  reference implementation only. EasyUse Anima should own its runtime code,
  class ids, storage keys, and data contracts.
- Keep NAIA out of the new regional node.
- Keep user-facing docs English-first for this stage. Runtime node text and
  custom DOM labels should still be managed through locale files and JS text
  maps where the UI needs localized labels.
- Include the NAIA resolution scaling settings for Advanced Prompt Studio.
- Include the image upscale-by-multiple helper node.

## Current Decisions

- New node candidate:
  - class id: `EasyUseAnimaPromptStudioRegional`
  - display name: `Anima Prompt Studio Regional`
  - category: `EasyUse Anima/Prompt`
- The node is a prompt planning and serialization node. It should not sample,
  patch a model, or encode CLIP conditionings directly.
- The node outputs metadata prompt strings, width, height, and dedicated
  regional data. `Anima Regional Conditioning` converts the regional data into
  KSampler-ready positive and negative conditionings.
- The node should not include NAIA request flags, NAIA resolution, or NAIA
  one-shot metadata logic.
- Saved-image workflow reload must reproduce the node state. Store the primary
  regional editor JSON in both `widgets_values` and a workflow property backup,
  following the existing Advanced Prompt Studio metadata pattern.
- Artist tags remain artist-category data. Do not add a runtime fallback that
  treats general tags as artist tags.
- Ordinary prompt tags should use spaces in final prompt text, with Pony score
  tags as the underscore-preserving exception.

## Structured Data Contract

`regional_prompt_data` should be a compact JSON object with these top-level
keys:

```json
{
  "version": 1,
  "canvas": {
    "width": 1024,
    "height": 1024,
    "aspect_ratio": "1:1",
    "source": "resolution_fields"
  },
  "mask_authoring": {
    "render_space": "image_pixels",
    "storage_space": "normalized_canvas",
    "preview_enabled": true
  },
  "global_prompt": "",
  "negative_prompt": "",
  "next_mask_id": 1,
  "masks": [],
  "regional_enabled": false,
  "mask_prompts": [],
  "assignments": [],
  "artist_mix": {},
  "conditioning_settings": {},
  "regional_settings": {}
}
```

`model_patch_data` should be a compact JSON object with these top-level keys:

```json
{
  "version": 1,
  "regional_attention": {},
  "layout_control": {},
  "global_mod_guidance": {},
  "artist_mix": {},
  "compatibility": {}
}
```

Initial rules:

- Canvas `width` and `height` are resolved from the Studio resolution fields.
  The mask editor should not introduce a separate mask resolution.
- Mask authoring renders against the resolved image size. The UI may display a
  scaled viewport, but preview and runtime mask generation must target the
  selected output width and height.
- Mask geometry should be stored as normalized canvas data, or with enough
  source canvas metadata to rescale safely when resolution changes.
- Masks are stored as structured numbered mask data, shapes, strokes, and
  settings inside `regional_config`; generated mask images are runtime outputs,
  not serialized image bodies.
- `regional_config.masks` is the canonical mask list. Each mask has a stable
  numeric `mask_id`, visible label, name, color, enabled state, geometry, and
  optional stroke data.
- New masks use `next_mask_id`. Deleting a mask must not silently renumber
  existing `mask_id` values because prompt fields may reference them.
- Prompt field assignments use `mask_ids`. The regional runtime may still call
  them regions when building conditioning, but the UI/storage selector is mask
  based.
- A prompt field with `mask_ids` applies only inside the selected user-authored
  masks.
- A prompt field with no `mask_ids` is global and applies to the full image.
- Multiple `mask_ids` on one field mean the prompt applies to the union of
  those masks.
- Mask-scoped prompts must remain structured in `regional_prompt_data`; do not
  flatten them into plain positive text before the conditioning/model-patch
  companion nodes read the mask assignments.
- Negative prompt fields are global in the initial implementation. Mask-scoped
  negative prompts may be added later only if the conditioning contract is
  explicit.
- `global_mod_guidance` is global, not per-region.
- Per-region Mod Guidance is out of the initial scope.
- Regional model patching may be stacked after global Mod Guidance:
  `MODEL -> global Mod Guidance -> Regional Model Patch -> sampler`.
- The KSampler still receives normal positive and negative conditionings.
- Model patch data should not contain image body data, data URLs, temp preview
  URLs, local personal paths, tokens, or API keys.

## Companion Node Candidates

The Prompt Studio Regional node should not perform all runtime work itself.
These companion nodes are planned or candidates:

### Anima Regional Conditioning

Purpose:

- Convert `CLIP + regional_prompt_data` into positive/negative conditioning.
- Apply mask-scoped prompt fields only inside their selected mask areas.
- Return a union mask and optional preview image where practical.
- Preserve global positive/negative compatibility with normal KSampler paths.

Candidate outputs:

| Name | Type | Purpose |
| --- | --- | --- |
| `positive` | `CONDITIONING` | Conditioning with regional metadata when regional prompting is enabled. |
| `negative` | `CONDITIONING` | Negative conditioning. |

Current 0.2.0 outputs are limited to `positive` and `negative`. Union mask,
mask preview, and pass-through sockets remain candidates for a later pass.

### Anima Regional Model Patch

Purpose:

- Apply regional attention, layout control, and artist-mix-aware model patching.
- Read `model_patch_data` rather than reading Prompt Studio UI state directly.
- Compose with the standalone `Anima Mod Guidance` model patch when possible.

Candidate inputs:

| Name | Type | Purpose |
| --- | --- | --- |
| `model` | `MODEL` | Model to patch. |
| `clip` | `CLIP` | Required only for patch modes that need text encoding. |
| `positive` | `CONDITIONING` | Positive conditioning used by compatible patch modes. |
| `negative` | `CONDITIONING` | Negative conditioning used by compatible patch modes. |
| `model_patch_data` | `STRING` JSON | Structured regional and artist mix patch data. |

Candidate outputs:

| Name | Type | Purpose |
| --- | --- | --- |
| `model` | `MODEL` | Patched model. |

## UI Behavior

- Start from the existing Prompt Studio Advanced DOM-widget storage model.
- Add a popup/modal mask editor for numbered masks:
  - rectangular split presets
  - grid presets
  - rectangle operations
  - brush and erase strokes only if the implementation remains lightweight
  - per-mask id, label, name, enabled state, and color
- The main node body should stay compact and expose a mask summary plus an Edit
  Masks button. Full mask editing happens in the popup/modal editor.
- The mask editor canvas uses the current resolved Studio resolution:
  - preset bucket/custom width and height define the mask authoring size
  - the visible canvas scales to fit the node or modal while preserving aspect
    ratio
  - pointer coordinates are converted through the canvas transform, stored as
    normalized mask data, and rendered to the selected image-pixel size
  - changing resolution re-renders the preview from stored mask data and
    should preserve normalized shapes where possible
- Include a mask preview in the editor:
  - show mask overlays with per-mask colors and alpha
  - show the active mask and union mask preview
  - show prompt-field assignment feedback when a field has selected
    `mask_ids`
  - do not serialize preview bitmaps, data URLs, or temporary preview paths
- Prompt fields are added, removed, reordered, resized, collapsed, and serialized
  like the existing Prompt Studio Advanced fields.
- Each positive prompt field has its own mask selector next to the prompt
  editor.
- The selector supports global/no mask, one mask, or multiple masks.
- Store the selected mask ids on that field as `mask_ids` inside
  `regional_fields`.
- When the mask list changes, selector options update from `regional_config`
  and preserved ids stay selected when possible. Missing ids should be shown as
  unassigned/invalid instead of being silently dropped.
- Negative sections are global by default.
- Artist sections are separate from general sections.
- The editor must keep hidden widgets serialized and must not remove required
  widgets from `node.widgets`.
- Layout must use one scheduled layout path and avoid render/layout/resize loops.

## Artist Mix Plan

Artist mix is part of the 0.2.0 scope.

Initial behavior:

- Artist sections produce artist prompt data.
- Artist prompt form is `@artist name`.
- Artist mix data is stored under both `regional_prompt_data.artist_mix` and
  `model_patch_data.artist_mix` when enabled.
- The first implementation should support disabled and average/exact-style
  modes only after the CLIP conditioning contract is clear.

Exclusions:

- Do not infer artists from general tags.
- Do not silently rescue bad CSV category data at runtime.
- Do not change the global prompt/tag normalization policy for non-artist tags.

## Mod Guidance And Regional Prompting

Existing standalone Mod Guidance is global. It reads `quality_tags`,
`positive`, and `negative` conditionings and patches `MODEL -> MODEL`.

0.2.0 should support composition in this order:

```text
metadata_prompt / metadata_negative_prompt -> metadata or optional global prompt path
CLIP + regional_prompt_data -> Anima Regional Conditioning -> KSampler
MODEL -> Anima Mod Guidance -> KSampler
nested model_patch_data -> future Anima Regional Model Patch
```

Initial exclusions:

- No per-region Mod Guidance in the first implementation.
- No sampler replacement in the first implementation.
- No hidden dependency on Spectrum KSampler internals from Prompt Studio
  Regional.

## Storage And Reproducibility

Required storage behavior:

- Hidden JSON widgets remain serialized.
- `onSerialize` must write current DOM state into `widgets_values`.
- Backend execution must mirror current JSON state into `EXTRA_PNGINFO`
  workflow metadata using `unique_id`.
- The workflow property backup should use a new key, for example
  `easyuse_anima_regional_fields`, not the existing Advanced Prompt Studio key.
- Saved-image workflow reload must restore the current regional fields and
  regional config, not a previous queued state.
- Add regression tests around `widgets_values`, workflow properties, and
  prompt metadata for the new node before broad UI work.
