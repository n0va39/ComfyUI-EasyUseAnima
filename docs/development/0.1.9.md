# 0.1.9 Development Plan

This file tracks changes intended for the next package version. It is not a
release announcement.

## Scope

- Use `0.1.9` as the next pre-release package marker after the 0.1.8 release.
- Include confirmed bug fixes from 0.1.7 and 0.1.8 behavior.
- Add wildcard functionality for Prompt Studio Advanced and a standalone string
  expansion node.
- Clean duplicated workflow documentation storage.
- Split user-facing node details out of the top-level README files.

## Current Decisions

- Public workflow JSON templates live only under `example_workflows/`.
- `docs/workflows/` is removed to avoid maintaining duplicate workflow JSON.
- Readable AiO workflow documentation stays under `docs/Anima AiO/`.
- General new-conversation entry order lives in `docs/development/README.md`;
  this file only tracks 0.1.9-specific scope.
- Wildcard work must not revive the abandoned 0.1.6 plan to reimplement Impact
  Pack `DetailerForEach` core.
- 0.1.9 wildcard support targets `Anima Prompt Studio Advanced` and
  `Anima Wildcard`. Prompt Builder, base Prompt Studio, Prompt Studio Extend,
  and LoRA Preset trigger conversion are deferred unless explicitly reopened.
- User wildcard files live under the ComfyUI user data folder. In the current
  v0.25.1 runtime this resolves to `ComfyUI/user/__easyuse_anima/wildcards`.
- Existing user-managed wildcard folders can be registered through
  `wildcard.extra_paths`.
- Wildcard expansion happens during node execution, not as edit-time preview.
- Saved-image workflows store expanded text for reproduction. Live workflows keep
  source wildcard text and the next seed state.
- Sequential mode is an EasyUse Anima extension. It uses
  `seed % candidate_count` and forces seed-after-generate to `increment`.
- `<lora:...>` syntax is preserved as text. 0.1.9 does not add MODEL/CLIP LoRA
  application from wildcard expansion.
- `MAINTAINING.md` still needs a policy sync only if development-guide changes
  are explicitly approved.

## Implemented Changes

- Fixed Prompt Studio Advanced NAIA resolution mode so repeated live queue runs
  can keep requesting fresh NAIA dimensions while saved-result workflows store
  the resolved dimensions as `Custom`.
- Fixed Prompt Studio highlight overlay alignment for browsers where the overlay
  and input caret could diverge.
- Added `wildcard_engine.py` with Impact Pack-oriented syntax support:
  - `__name__`, `__*/name__`, `__folder/*__`, and `N#__name__`
  - `{a|b|c}`
  - weighted dynamic options such as `{2::a|5::b|c}`
  - multi-select syntax such as `{2$$a|b|c}` and `{1-3$$, $$a|b|c}`
  - `.txt`, `.yaml`, and `.yml` wildcard files
  - recursive expansion depth limit
- Added default wildcard folder creation with `easyuse_anima_test.txt`.
- Added `Anima Wildcard` node.
- Added Prompt Studio Advanced wildcard controls below `mod guidance`.
- Added wildcard autocomplete API and `__` input autocomplete.
- Added separate Prompt Studio wildcard highlighting.
- Added `wildcard.extra_paths` to EasyUse Anima settings.
- Added tests for wildcard expansion, default folder creation, extra path
  priority, sequential selection, autocomplete-safe key listing, public settings,
  and saved workflow reproduction metadata.
- Added user-facing wildcard documentation:
  - `docs/wildcards.ko.md`
  - `docs/wildcards.en.md`
- Added per-node user documentation under `docs/nodes/` and moved README node
  details to that entry point.
- Added node screenshots under `docs/images/nodes/`.

## Deferred Scope

- Wildcard controls for Prompt Builder, base Prompt Studio, and Prompt Studio
  Extend.
- LoRA Preset trigger-to-wildcard conversion.
- Edit-time wildcard preview or a dedicated wildcard picker UI.
- MODEL/CLIP/CONDITIONING wildcard nodes that apply `<lora:...>` syntax.
- Impact Pack detailer wildcard features such as `[LAB]`, `[ASC]`, `[DSC]`,
  `[RND]`, and `[SEP]`.
- Comfy Registry publish and release tag creation.
