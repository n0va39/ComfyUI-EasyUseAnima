# 0.1.9 Development Plan

This file tracks changes intended for the next package version. It is not a
release announcement.

## Scope

- Use `0.1.9` as the next pre-release package marker after the 0.1.8 release.
- Include confirmed bug fixes from 0.1.7 and 0.1.8 behavior.
- Add wildcard functionality.
- Clean duplicated workflow documentation storage.

## Current Decisions

- Public workflow JSON templates live only under `example_workflows/`.
- `docs/workflows/` is removed to avoid maintaining duplicate workflow JSON.
- Readable AiO workflow documentation stays under `docs/Anima AiO/`.
- General new-conversation entry order lives in `docs/development/README.md`;
  this file only tracks 0.1.9-specific scope.
- Wildcard work must not revive the abandoned 0.1.6 plan to reimplement Impact
  Pack `DetailerForEach` core unless that scope is explicitly reopened.
- Exact wildcard UI/runtime behavior still needs confirmation before implementation.
- `MAINTAINING.md` still needs a policy sync if the user approves development
  guide changes.

## Review Order For 0.1.9

1. Confirm the docs/workflow cleanup diff first.
2. Confirm the 0.1.9 version marker and workflow metadata alignment.
3. Identify concrete 0.1.7 and 0.1.8 regressions before implementing fixes.
4. Define wildcard syntax, storage location, expansion timing, and seed behavior.
5. Add focused tests for each fixed regression before broader wildcard work.
6. Only then implement wildcard UI/runtime changes.

## Missing Decisions Before Wildcard Implementation

- Wildcard syntax: file-backed `__name__`, inline `{a|b}`, both, or another
  format.
- Wildcard source directory and whether user-defined files are allowed.
- Expansion timing: edit-time preview, queue-time expansion, or node execution.
- Randomness contract: deterministic by seed, per-run random, or per-cycle.
- Compatibility target: Prompt Studio only, Prompt Studio Advanced, LoRA Preset
  triggers, or all prompt-producing nodes.
- Serialization strategy for saved workflows and shared workflows.

## Planned Changes

- Fix confirmed 0.1.7 and 0.1.8 regressions as they are identified.
- Fix Prompt Studio Advanced NAIA resolution mode so the live workflow keeps
  requesting fresh NAIA dimensions on repeated queue runs while saved result
  workflows store the resolved dimensions as `Custom` for reproducibility.
- Add wildcard parsing and expansion support with deterministic queue-time
  behavior.
- Preserve saved workflow compatibility when adding wildcard-related inputs or
  widgets.
- Add tests for wildcard expansion, empty or malformed wildcard text, and saved
  workflow compatibility.
- Keep workflow template package metadata aligned with package version `0.1.9`.
- Keep release workflow JSON in `example_workflows/` only.
