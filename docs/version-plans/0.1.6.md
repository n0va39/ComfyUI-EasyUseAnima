## 상태

- [ ] 계획 검토 완료
- [ ] 구현 시작
- [ ] 기능 구현 완료

## 목표

0.1.6에서는 다음 작업을 진행한다.

- 자동완성 동작 범위를 사용자가 선택할 수 있게 한다.
- 자동완성과 프롬프트 교정기가 같은 태그 정규화/파싱 규칙을 사용하도록 정리한다.
- 자동완성 CSV 데이터의 역할, 선택 기준, 사용 방법을 가이드 문서에 정리한다.
- Anima Prompt Studio Advanced의 입력창 높이와 스크롤 정책을 다시 정리한다.
- EasyUse Anima 자체 언어 설정을 제거하고, 모든 UI 문구를 ComfyUI 기본 언어 설정에 맞춰 표시한다.
- Impact Pack의 디테일러 구조와 참고용 워크플로우의 노드 구성을 보고 ANIMA용 디테일러 노드 설계를 시작한다.

ANIMA 디테일러는 범위가 크므로 한 번에 완성하지 않는다. `docs/workflows/Anima_AiO_v5.5.1.json`은 참고용 워크플로우이며, 아래 노드 번호는 그 파일 안에서 구현 참고 대상을 찾기 위한 기준이다. 이번 범위에서는 `MaskToSEGS` 대체 노드는 만들지 않고, Impact Pack `DetailerForEach`의 원본 구조를 파악한 뒤 구조 변경이 필요 없는 부분은 동일하게 구현한다. 핵심은 참고 워크플로우에서 쓰는 최적화 노드와 optional 입력이 켜져 있어도 오류가 나지 않는 ANIMA용 Detailer를 만드는 것이다.

## 1. 자동완성 설정

### 목표

EasyUse Anima 자동완성이 어디에서 동작할지 설정할 수 있게 한다.

### 설정 모드

- [ ] `off`
  - EasyUse Anima 자동완성을 전부 끈다.
  - 입력 이벤트 핸들러를 붙이지 않는다.
  - 자동완성 팝업을 열지 않는다.
  - `/easyuse_anima/autocomplete` API를 호출하지 않는다.

- [ ] `easyuse_nodes`
  - EasyUse Anima 전용 프롬프트 노드에서만 자동완성을 사용한다.
  - 대상:
    - Anima Prompt Studio
    - Anima Prompt Studio Advanced
    - 이후 추가되는 EasyUse Anima 프롬프트 편집 노드
  - 일반 ComfyUI 문자열 입력 위젯에는 붙이지 않는다.

- [ ] `compatible_global`
  - 현재 방식이다.
  - 지원 가능한 일반 텍스트 위젯에도 자동완성을 붙인다.
  - LoRA Manager, LoRA Stacker 등 자체 자동완성이나 LoRA 전용 입력을 가진 노드는 계속 제외한다.

### 기본값

- [ ] 기본값은 `compatible_global`로 둔다.
- [ ] 기존 사용자 동작이 갑자기 바뀌지 않게 한다.

### 구현 방법

- [ ] 설정 키를 추가한다.
  - 후보: `autocomplete.mode`
- [ ] `web/js/easyuse_anima_settings.js`에 설정 UI를 추가한다.
  - 기존 자동완성 CSV 선택과 결과 개수 설정 근처에 배치한다.
- [ ] `web/js/easyuse_anima_autocomplete.js`에 모드 판정 함수를 추가한다.
  - 후보 함수:
    - `autocompleteMode()`
    - `isEasyUseAnimaAutocompleteTarget(widgetOrElement, node)`
    - `shouldAttachEasyUseAutocomplete(widgetOrElement, node)`
- [ ] 자동완성 핸들러를 붙이기 전에 설정을 확인한다.
- [ ] 팝업을 열기 전에 설정을 확인한다.
- [ ] API 요청 전에 설정을 확인한다.
- [ ] `off`일 때 이미 열린 팝업이 있으면 닫는다.
- [ ] 설정 변경 이벤트를 받아 브라우저 새로고침 없이 가능한 범위에서 반영한다.
- [ ] `compatible_global`에서는 기존 제외 규칙을 유지한다.

### 검증 기준

- [ ] `off`에서 자동완성 팝업이 뜨지 않는다.
- [ ] `off`에서 `/easyuse_anima/autocomplete` 요청이 발생하지 않는다.
- [ ] `easyuse_nodes`에서 EasyUse Anima 프롬프트 노드에서는 자동완성이 동작한다.
- [ ] `easyuse_nodes`에서 일반 문자열 입력 위젯에는 자동완성이 붙지 않는다.
- [ ] `compatible_global`에서 기존 방식처럼 동작한다.
- [ ] LoRA Manager / Lora Stacker 계열 입력창에는 EasyUse Anima 자동완성이 뜨지 않는다.
- [ ] `node --check web/js/easyuse_anima_autocomplete.js` 통과
- [ ] `node --check web/js/easyuse_anima_settings.js` 통과

  - 한국어 검색 지원 데이터
  - 태그 분류와 하이라이트 보조 데이터

- [ ] 설정창에서 자동완성 CSV를 선택하는 위치를 설명한다.

- [ ] 사용자 CSV 추가 조건이 문서에 명확히 적혀 있다.

## 3. 자동완성과 프롬프트 교정 일관성

### 목표

자동완성에서 선택되는 태그와 프롬프트 교정기가 최종 출력하는 태그가 서로 다른 문법으로 바뀌지 않게 한다.

### 구현 원칙

- [ ] 자동완성 삽입, 하이라이트 분류, 프롬프트 교정이 같은 태그 파싱 helper를 사용하게 한다.
- [ ] 일반 Danbooru/meta/artist 태그는 공백 표기를 기본으로 한다.
- [ ] Pony score 태그는 예외로 언더바를 유지한다.
  - `score_9`
  - `score_8`
  - `score_7`
  - `score_6`
  - `score_5`
  - `score_4`
  - `score_7:` 같은 colon variant
  - `(score_8:0.65)` 같은 weighted variant
- [ ] 수동 트리거 입력은 그 자체로 보존한다.
- [ ] LoRA 트리거/수동 override 영역은 프롬프트 교정기가 임의로 언더바를 공백으로 바꾸지 않는다.
- [ ] 작가 태그는 `@artist name` 형태를 기준으로 하고, 자동완성과 교정 결과가 서로 다르게 바뀌지 않게 한다.
- [ ] 괄호 가중치 문법은 태그 판정에서 괄호와 숫자를 분리해 처리한다.
- [ ] 괄호가 닫히지 않은 문법 오류는 오타 밑줄과 구분해서 표시한다.

### 검증 기준

- [ ] 자동완성으로 삽입한 태그를 프롬프트 교정기에 넣어도 의미 없는 재변환이 발생하지 않는다.
- [ ] `score_8`, `score_7:` 등 Pony score 태그의 언더바가 유지된다.
- [ ] 일반 태그와 작가 태그는 불필요한 언더바가 생기지 않는다.
- [ ] 수동 트리거 텍스트는 입력 그대로 유지된다.
- [ ] weighted prompt 안의 태그와 가중치 숫자가 각각 올바르게 분리되어 표시된다.
- [ ] 회귀 테스트에 Pony score, 작가 태그, 수동 트리거, weighted prompt 사례를 포함한다.

## 4. Advanced 입력창 스크롤/높이 정책

### 목표

Anima Prompt Studio Advanced에서 개별 입력창의 스크롤바가 글자를 가리는 문제를 없애고, 긴 프롬프트는 노드 내부의 큰 편집 영역에서 스크롤되게 한다.

### 구현 원칙

- [ ] 개별 textarea에는 세로 스크롤바가 생기지 않게 한다.
- [ ] 개별 textarea 높이는 현재 텍스트 줄 수를 기준으로 자동 확장한다.
- [ ] 개별 textarea는 현재 텍스트를 모두 보여주는 최소 높이보다 작아질 수 없게 한다.
- [ ] 기본 빈 입력창 높이는 2줄 정도로 둔다.
- [ ] 사용자가 textarea 높이를 수동으로 늘릴 수는 있지만, 텍스트 줄 수 기준 최소 높이보다 줄일 수는 없게 한다.
- [ ] 개별 입력창들이 길어져 노드 표시 영역을 넘어가면, 개별 textarea가 아니라 Advanced 편집 영역 전체가 스크롤된다.
- [ ] 노드 전체가 무한히 길어지지 않도록 editor viewport 높이와 content 높이를 분리한다.
- [ ] 내부 전체 스크롤 영역은 입력창, 버튼, 범례, mod guidance, 해상도 버켓 등 모든 고정 UI 높이를 포함해 계산한다.
- [ ] 노드 resize 중에는 강제 재배치를 반복하지 않고, resize 종료 후 editor viewport 높이만 보정한다.
- [ ] textarea 높이 저장값과 노드 전체 높이 저장값을 분리한다.

### 검증 기준

- [ ] 긴 텍스트 입력 시 개별 textarea에 내부 스크롤바가 생기지 않는다.
- [ ] 긴 텍스트 입력 시 textarea 높이가 줄 수에 맞춰 늘어난다.
- [ ] textarea 높이를 줄여도 텍스트 줄 수보다 낮게 줄어들지 않는다.
- [ ] 입력창들이 많거나 길어지면 Advanced 편집 영역 전체에 스크롤이 생긴다.
- [ ] 스크롤바가 텍스트 위를 덮거나 글자를 가리지 않는다.
- [ ] 노드 resize 후 입력창과 버튼이 겹치지 않는다.
- [ ] workflow 저장/불러오기 후 textarea 높이와 editor 스크롤 정책이 유지된다.

## 5. ComfyUI 기본 언어 기반 UI 문구 관리

### 목표

ComfyUI 공식 embedded-docs의 언어별 문서 구조처럼, EasyUse Anima의 정보 패널, 가이드, 힌트, 툴팁, 설정 설명 등 모든 UI 문구를 ComfyUI 기본 언어 설정에 맞춰 표시한다.

### 구현 방법

- [ ] EasyUse Anima 설정창에서 자체 언어 설정을 제거한다.
- [ ] EasyUse Anima는 별도 언어 상태를 저장하지 않는다.
- [ ] 기존 설정 파일에 남은 EasyUse Anima 언어 설정 값이 있으면 읽지 않거나 마이그레이션으로 제거한다.
- [ ] 모든 UI 문구는 EasyUse Anima 자체 언어 설정이 아니라 ComfyUI 기본 언어 설정을 따른다.
- [ ] ComfyUI 프론트엔드에서 기본 언어 설정을 읽는 공식 경로 또는 안정적인 내부 경로를 확인한다.
- [ ] ComfyUI 기본 언어가 한국어이면 한국어 문구를 표시한다.
- [ ] ComfyUI 기본 언어가 한국어가 아니면 영어 문구를 표시한다.
- [ ] 영어 문구를 기본 fallback으로 둔다.
- [ ] 정보 패널, 설정창, 가이드 텍스트, 힌트, 툴팁 문구가 흩어지지 않게 공통 i18n helper 또는 공통 text map을 만든다.
- [ ] 각 UI 렌더링 시 ComfyUI 기본 언어 설정을 읽어 해당 언어 문구를 선택한다.
- [ ] ComfyUI 언어 설정 변경 이벤트가 확인되면 가능한 범위에서 UI 문구를 즉시 갱신한다.
- [ ] ComfyUI 자체 설정 목록 이름처럼 즉시 갱신이 어렵거나 ComfyUI 내부가 관리하는 영역은 구현 한계로 문서화한다.

### 문구 관리 원칙

- [ ] 모든 UI 문구는 언어별로 같은 키를 가진 구조로 관리한다.
- [ ] 정보 패널 노드 설명은 가능하면 고정 섹션을 사용한다.
- [ ] 노드별 설명 구조는 가능하면 고정한다.
  - 개요
  - 입력
  - 출력
  - 주요 옵션
  - 참고 / 제한사항
- [ ] 가이드/힌트/툴팁 문구도 같은 i18n 구조에서 관리한다.
- [ ] 한국어 문구는 ComfyUI 한국어 사용자 기준으로 작성한다.
- [ ] 영어 문구와 한국어 문구가 서로 다른 기능 설명을 하지 않게 한다.
- [ ] 누락된 번역 키가 있으면 영어 문구로 fallback한다.

### 원본 구조 확인 결과

Impact Pack `DetailerForEach`는 다음 구조를 가진다.

- required 입력:
  - `image: IMAGE`
  - `segs: SEGS`
  - `model: MODEL`
  - `clip: CLIP`
  - `vae: VAE`
  - `guide_size`
  - `guide_size_for`
  - `max_size`
  - `seed`
  - `steps`
  - `cfg`
  - `sampler_name`
  - `scheduler`
  - `positive: CONDITIONING`
  - `negative: CONDITIONING`
  - `denoise`
  - `feather`
  - `noise_mask`
  - `force_inpaint`
  - `wildcard`
  - `cycle`
- optional 입력:
  - `detailer_hook: DETAILER_HOOK`
  - `inpaint_model`
  - `noise_mask_feather`
  - `scheduler_func_opt: SCHEDULER_FUNC`
  - `tiled_encode`
  - `tiled_decode`
- 출력:
  - `IMAGE`
- 핵심 처리:
  - image batch는 허용하지 않는다.
  - `segs_scale_match`로 SEGS와 image shape를 맞춘다.
  - segment별 crop region을 가져와 detail inpaint를 실행한다.
  - conditioning에 mask가 있으면 segment crop region 기준으로 mask를 crop한다.
  - wildcard는 segment별로 처리한다.
  - `detailer_hook`이 있으면 크기, encode/decode, sampler, paste 단계에 hook을 적용한다.
  - `scheduler_func_opt`, `tiled_encode`, `tiled_decode`는 `enhance_detail`로 그대로 전달한다.

### 설계 원칙

- [ ] 원본 `DetailerForEach`와 같은 입출력 구조를 유지한다.
- [ ] 구조 변경이 필요 없는 입력, optional 입력, 출력은 원본과 동일하게 둔다.
- [ ] `MaskToSEGS` 대체 노드는 이번 범위에서 만들지 않는다.
- [ ] `SEGS`는 외부 노드에서 들어온 값을 그대로 받는다.
- [ ] `DetailerForEach.do_detail` 흐름을 기준으로 구현한다.
- [ ] detailer hook과 scheduler function은 비활성/미연결/활성/연결 상태 모두에서 오류가 없어야 한다.
- [ ] workflow에서 UE/위젯 연결 기능으로 각 파라미터가 외부 연결되어도 queue validation이 깨지지 않아야 한다.
- [ ] `tiled_encode`, `tiled_decode`, `scheduler_func_opt`, `detailer_hook` 같은 최적화 입력은 꺼져 있으면 원본 기본 동작을 유지하고, 켜져 있으면 값만 안전하게 전달한다.
- [ ] ANIMA 전용 변경은 크기 정렬, VAE 호환성, 오류 메시지, 기본값 보강에 한정한다.
- [ ] 원본 구조를 무리하게 단순화해서 workflow 호환성을 깨지 않는다.

### 최적화 노드 호환성 목표

참고 워크플로우에서 최적화 관련 입력이 켜져 있거나 외부 노드와 연결되어도 새 Detailer가 실패하지 않게 한다.

- [ ] `detailer_hook`이 `None`이어도 동작한다.
- [ ] `detailer_hook`이 연결되어 있으면 원본처럼 hook 메서드 호출 지점이 유지된다.
- [ ] `scheduler_func_opt`가 `None`이어도 동작한다.
- [ ] `scheduler_func_opt`가 연결되어 있으면 sampler 단계로 전달된다.
- [ ] `tiled_encode`/`tiled_decode`가 켜져 있어도 VAE encode/decode 경로에서 오류가 나지 않는다.
- [ ] `inpaint_model`, `noise_mask_feather`가 켜져도 원본의 differential diffusion 적용 조건을 유지한다.
- [ ] 외부 primitive/control 노드가 `guide_size`, `max_size`, `steps`, `cfg`, `denoise`, `wildcard` 등을 연결해도 입력 타입과 기본값이 깨지지 않는다.
- [ ] 최적화 입력을 지원하지 못하는 경우 조용히 무시하지 말고 어느 입력이 미지원인지 명확히 오류를 낸다.

## 8. 16채널 VAE와 크기 보정

문제:

- [ ] 모든 crop/detail 크기는 안전한 배수로 정렬한다.
- [ ] 우선 32픽셀 배수 정렬을 기본 후보로 둔다.

- [ ] tiled encode/decode가 필요한 경우 숨기지 않고 옵션으로 드러낸다.
- [ ] 크기를 자동 조정했다면 로그나 report로 알 수 있게 한다.

구현 체크리스트:

- [ ] Impact Pack에서 crop region, guide size, max size를 계산하는 위치를 확인한다.
- [ ] ANIMA용 크기 정렬 helper를 만든다.
- [ ] helper 입력을 정한다.
  - crop width
  - crop height
  - guide size
  - max size
  - alignment
- [ ] helper 출력을 정한다.
  - safe width
  - safe height
  - scale decision
  - adjusted 여부
- [ ] guide size/max size를 직접 바꿀지, 최종 crop 크기만 보정할지 결정한다.
- [ ] ComfyUI 런타임 없이 테스트 가능한 단위 테스트를 추가한다.
