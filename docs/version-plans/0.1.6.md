## 1. 자동완성 설정

### 목표

EasyUse Anima 자동완성이 어디에서 동작할지 설정할 수 있게 한다.

### 설정 모드

- [x] `off`
  - EasyUse Anima 자동완성을 전부 끈다.
  - 이미 붙은 입력 이벤트 핸들러는 팝업과 API 호출 없이 비활성 상태로 둔다.
  - 자동완성 팝업을 열지 않는다.
  - `/easyuse_anima/autocomplete` API를 호출하지 않는다.

- [x] `easyuse_nodes`
  - EasyUse Anima 전용 프롬프트 노드에서만 자동완성을 사용한다.
  - 대상:
    - Anima Prompt Studio
    - Anima Prompt Studio Advanced
    - 이후 추가되는 EasyUse Anima 프롬프트 편집 노드
  - 일반 ComfyUI 문자열 입력 위젯에는 붙이지 않는다.

- [x] `compatible_global`
  - 현재 방식이다.
  - 지원 가능한 일반 텍스트 위젯에도 자동완성을 붙인다.
  - LoRA Manager, LoRA Stacker 등 자체 자동완성이나 LoRA 전용 입력을 가진 노드는 계속 제외한다.

### 기본값

- [x] 기본값은 `compatible_global`로 둔다.
- [x] 기존 사용자 동작이 갑자기 바뀌지 않게 한다.

### 구현 방법

- [x] 설정 키를 추가한다.
  - 후보: `autocomplete.mode`
- [x] `web/js/easyuse_anima_settings.js`에 설정 UI를 추가한다.
  - 기존 자동완성 CSV 선택과 결과 개수 설정 근처에 배치한다.
- [x] `web/js/easyuse_anima_autocomplete.js`에 모드 판정 함수를 추가한다.
  - 후보 함수:
    - `autocompleteMode()`
    - `isEasyUseAnimaAutocompleteTarget(widgetOrElement, node)`
    - `shouldAttachEasyUseAutocomplete(widgetOrElement, node)`
- [x] 자동완성 핸들러 실행 전에 설정을 확인한다.
- [x] 팝업을 열기 전에 설정을 확인한다.
- [x] API 요청 전에 설정을 확인한다.
- [x] `off`일 때 이미 열린 팝업이 있으면 닫는다.
- [x] 설정 변경 이벤트를 받아 브라우저 새로고침 없이 가능한 범위에서 반영한다.
- [x] `compatible_global`에서는 기존 제외 규칙을 유지한다.

### 검증 기준

- [ ] `off`에서 자동완성 팝업이 뜨지 않는다.
- [ ] `off`에서 `/easyuse_anima/autocomplete` 요청이 발생하지 않는다.

- [ ] LoRA Manager / Lora Stacker 계열 입력창에는 EasyUse Anima 자동완성이 뜨지 않는다.

## 3. 자동완성과 프롬프트 교정 일관성

### 목표

자동완성에서 선택되는 태그와 프롬프트 교정기가 최종 출력하는 태그가 서로 다른 문법으로 바뀌지 않게 한다.

### 구현 원칙

- [x] 자동완성 삽입, Prompt Studio 미리보기, 프롬프트 교정 결과가 같은 표기 규칙을 따르게 한다.
- [x] 일반 Danbooru/meta/artist 태그는 공백 표기를 기본으로 한다.
- [x] Pony score 태그는 예외로 언더바를 유지한다.
  - `score_9`
  - `score_8`
  - `score_7`
  - `score_6`
  - `score_5`
  - `score_4`
  - `score_7:` 같은 colon variant
  - `(score_8:0.65)` 같은 weighted variant
- [x] 수동 트리거 입력은 그 자체로 보존한다.
- [x] LoRA 트리거/수동 override 영역은 프롬프트 교정기가 임의로 언더바를 공백으로 바꾸지 않는다.
- [x] 작가 태그는 `@artist name` 형태를 기준으로 하고, 자동완성과 교정 결과가 서로 다르게 바뀌지 않게 한다.
- [x] 괄호 가중치 문법은 태그 판정에서 괄호와 숫자를 분리해 처리한다.
- [x] 괄호가 닫히지 않은 문법 오류는 오타 밑줄과 구분해서 표시한다.

### 검증 기준

- [x] 자동완성으로 삽입한 태그를 프롬프트 교정기에 넣어도 의미 없는 재변환이 발생하지 않는다.
- [x] `score_8`, `score_7:` 등 Pony score 태그의 언더바가 유지된다.
- [x] 일반 태그와 작가 태그는 불필요한 언더바가 생기지 않는다.
- [x] 수동 트리거 텍스트는 입력 그대로 유지된다.
- [x] weighted prompt 안의 태그와 가중치 숫자가 각각 올바르게 분리되어 표시된다.
- [x] 회귀 테스트에 Pony score, 작가 태그, 수동 트리거, weighted prompt 사례를 포함한다.

## 4. Advanced 입력창 스크롤/높이 정책

### 구현 원칙

- [x] 개별 textarea에는 세로 스크롤바가 생기지 않게 한다.
- [x] 개별 textarea 높이는 현재 텍스트 줄 수를 기준으로 자동 확장한다.
- [x] 개별 textarea는 현재 텍스트를 모두 보여주는 최소 높이보다 작아질 수 없게 한다.
- [x] 기본 빈 입력창 높이는 2줄 정도로 둔다.
- [x] 사용자가 textarea 높이를 수동으로 늘릴 수는 있지만, 텍스트 줄 수 기준 최소 높이보다 줄일 수는 없게 한다.
- [x] 개별 입력창들이 길어져 노드 표시 영역을 넘어가면, 개별 textarea가 아니라 Advanced 편집 영역 전체가 스크롤된다.
- [x] 노드 전체가 무한히 길어지지 않도록 editor viewport 높이와 content 높이를 분리한다.
- [x] 내부 전체 스크롤 영역은 입력창, 버튼, 범례, mod guidance, 해상도 버켓 등 모든 고정 UI 높이를 포함해 계산한다.

- [x] textarea 높이 저장값과 노드 전체 높이 저장값을 분리한다.

- [x] textarea 높이를 줄여도 텍스트 줄 수보다 낮게 줄어들지 않는다.
- [x] 입력창들이 많거나 길어지면 Advanced 편집 영역 전체에 스크롤이 생긴다.
- [x] 스크롤바가 텍스트 위를 덮거나 글자를 가리지 않는다.
- [x] 노드 resize 후 입력창과 버튼이 겹치지 않는다.

## 5. ComfyUI 기본 언어 기반 UI 문구 관리

### 목표

ComfyUI 공식 embedded-docs의 언어별 문서 구조처럼, EasyUse Anima의 정보 패널, 가이드, 힌트, 툴팁, 설정 설명 등 모든 UI 문구를 ComfyUI 기본 언어 설정에 맞춰 표시한다.

### 구현 방법

- [x] EasyUse Anima 설정창에서 자체 언어 설정을 제거한다.
- [x] EasyUse Anima는 별도 언어 상태를 저장하지 않는다.
- [x] 기존 설정 파일에 남은 EasyUse Anima 언어 설정 값이 있어도 UI 문구 선택에 사용하지 않는다.
- [x] 정보 패널, 입력 이름, 입력 tooltip, 출력 이름, 출력 tooltip은 ComfyUI 공식 커스텀 노드 locale 구조인 `locales/<lang>/nodeDefs.json`로 관리한다.
- [x] Impact Pack처럼 `locales/ko/nodeDefs.json`을 추가하고, ComfyUI가 `/i18n`에서 자동 수집하도록 한다.
- [x] 파이썬 `DESCRIPTION`과 `OUTPUT_TOOLTIPS`는 영어 fallback으로 유지한다.
- [x] 설정창, 커스텀 DOM 버튼, canvas 버튼, alert/prompt 등 `nodeDefs.json`이 커버하지 않는 문구는 JS text map으로 관리한다.
- [x] JS text map은 ComfyUI 기본 언어가 한국어이면 한국어를, 그 외에는 영어를 선택한다.

## 6. 디테일러 훅 전환

### 결정

ANIMA용 DetailerForEach 대체 노드는 0.1.6 범위에서 취소한다. Impact Pack의 `DetailerForEach`를 그대로 사용하고, EasyUse Anima는 `DETAILER_HOOK`만 제공한다.

이유:

- Impact Pack detail loop, wildcard, scheduler function, tiled encode/decode, inpaint option을 재구현하면 유지보수 범위가 커진다.
- Impact Pack 내부 코드를 직접 복제하면 라이선스와 업데이트 추적 부담이 커진다.
- 실제 문제는 crop sampling 크기가 32배수가 아니어서 발생하는 latent size mismatch이므로, `touch_scaled_size()` hook만으로 해결하는 편이 안정적이다.

### 구현 결과

- [x] `Anima Detailer Align Hook` 노드를 추가했다.
- [x] `DETAILER_HOOK` 출력으로 Impact Pack `DetailerForEach`의 `detailer_hook` 입력에 연결할 수 있다.
- [x] 기본 정렬값은 `32`로 둔다.
- [x] `none`을 선택하면 Impact Pack 원본 크기를 유지한다.
- [x] 기존 `detailer_hook`이 연결되어 있으면 기존 hook을 먼저 실행한 뒤 크기 정렬만 적용한다.
- [x] `Anima Detailer` 공개 노드는 제거했다.
- [x] 디테일러 훅 추가 상태에서 참고 워크플로우의 최적화 노드들이 문제없이 실행되는 것을 확인했다.

### 유지할 공개 노드

- [x] `Anima Detailer Align Hook`
- [x] `Anima SAM3 Context`
- [x] ~~`Anima SAM3 Detailer` 정리~~ 계획 변경으로 0.1.6 범위에서 폐기한다.

### 제외할 공개 노드

- [x] `Anima Detailer`
  - Impact Pack `DetailerForEach`를 단순 재노출하는 wrapper이므로 유지하지 않는다.
  - 사용자는 Impact Pack `DetailerForEach`에 `Anima Detailer Align Hook`을 직접 연결한다.

## 8. 16채널 VAE와 크기 보정

- [x] 예: `1052 x 1232 -> 1056 x 1248`

- [x] Impact Pack의 기존 hook이 있으면 기존 hook 실행 후 크기 정렬만 추가한다.
